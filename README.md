В репозитории есть папки:

## Инструкция по запуску python_tasks

Необходимые зависимости описаны в файле ```requirements.txt```, поэтому для того, чтобы запускать программы, необходимо выполнить:

1. Установка и активация виртуального окружения

```
python -m venv .venv
.venv\Scripts\activate.bat
```

2. Установка зависимостей

```
pip install -r requirements.txt
```

Теперь можно запускать каждый из файлов с помощью команды:

 ```
 python путь_к_файлу.py
 ```

 Программы task06 и task07 выполняются иначе, их запуск описан в следующем блоке

## Описание программ 1-7

### task01_text_wrap

Эта программа выполняет переносы строк текста (не разбивая при этом слова), если в строке текста находится больше 70 символов.

Для примера используется текст самого задания:

```Дан текст. Напишите программу, которая отформатирует этот текст так, чтобы в строке текста было не более 70 символов, а потом шел перенос строки. Слова при этом не должны разбиваться.```

Он будет разбит программой на:

```
Дан текст. Напишите программу, которая отформатирует этот текст так, (перенос тут)
чтобы в строке текста было не более 70 символов, а потом шел перенос (перенос тут)
строки. Слова при этом не должны разбиваться.
```

### task02_lucky_ticket

Программа берёт 6 заданных чисел (потому что билет может состоять только из 6 чисел), и проверяет, что сумма первых трех и последних трех символов совпадает. Если да, возвращается True, если нет, то False.

### task03_nested_getter

Программа работает со словарём, который может содержать другие словари внутри себя. Функция принимает строку из ключей, которые разделяются точками, а после этого последовательно идёт по объекту и возвращает нужное значение, которое там и содержится.

### task04_json_parse

Программа получает строку JSON и преобразует ее в python-объект, который уже может просто использоваться как обычный словарь.

### task05_pandas_fillna

Программа работает с двумя датафреймами pandas. В первом датафрейме есть пустые значения NaN, и они должны быть заменены их соответствующими значениями из второго датафрейма. Дальше результат сохраняется уже в новый датафрейм df3.

### task06_fastapi_crud

Программа представляет собой простой API для управления пользователями.

У каждого юзера есть  ```id```, ```name``` и ```email```.

Доступны такие функции, как:

- создание пользователя
- получение списка пользователей
- получение пользователя по идентификатору
- обновление данных пользователя
- удаление пользователя

Для проверки работы API можно использовать встроенную документацию FastAPI, открыв в браузере http://127.0.0.1:8000/docs

Запуск, находясь в директории с файлом app.py:
```
uvicorn app:app --reload
```

### task07_fastapi_jwt

Программа реализует систему аутентификации с использованием jwt-токенов

Пользователь может зарегистрироваться, войти, получить jwt-токен, использовать токен, чтобы проходить по защищенным эндпоинтам.
Если вход в приложнеие успешный, то приложение выдаёт токен

Для доступа к защищенным эндпоинтам этот токен нужно передавать в заголовке запроса

Если токен действителен, пользователь получает доступ к защищенному маршруту. Иначе приложение вернет ошибку авторизации.

Запуск, находясь в директории с файлом app.py:

```
uvicorn app:app --reload
```

После запуска документация будет доступна по адресу http://127.0.0.1:8000/docs

Чтобы провеить работу, необходимо:

Сначала выполнить запрос на регистрацию пользователя через эндпоинт ```/register```.

Например с таким телом:

```
{
  "username": "user",
  "email": "email",
  "password": "pass"
}
```

Теперь выполнить запрос на вход через эндпоинт ```/login```.

В ответ будет получен JWT-токен, его нужно скопировать.

Дальше нажать кнопку ```Authorize``` в Swagger и вставить токен.
После этого, если мы выполним запрос к защищенному эндпоинту ```/protected```, мы получим сообщение об успешном доступе.
